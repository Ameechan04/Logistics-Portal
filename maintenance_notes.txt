Shipments Table:
The Status' (delivered, in transit, delayed, cancelled) and Service Types are hardcoded to reduce number of database calls.
Working on the assumption that future additions to the database will follow the pattern of the existing providers.
The Carrier is not hardcoded allowing scope for future expansion in the database to cover more carrier providers.

Homepage:
The HomePage component now handles its own data fetching. The fetchDashboardData function simulates an asynchronous API call that
accepts a filters object and returns the calculated metrics.
